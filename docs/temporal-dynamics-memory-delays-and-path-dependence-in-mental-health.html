Agent Based Modelling

Chapter 20 Temporal Dynamics: Memory, Delays, and Path Dependence in Mental Health

In earlier agent-based models, we saw how simple local rules could produce surprisingly rich collective outcomes. Random walks transformed aimless motion into diffusion, and Schelling’s segregation model showed how mild neighborhood preferences can generate sharply divided cities. Here, we turn to a different kind of emergence: not spatial patterning, but temporal patterning. The question is no longer where agents move, but how past stress lingers, accumulates, and shapes future vulnerability. In this model, mental health is not a static category but a dynamic process with memory, delayed effects, and path dependence.

That shift is important. Many real-world social and behavioral processes do not respond only to what is happening right now. They respond to what has been happening for days, weeks, or years. Stress exposure can accumulate. Recovery can lag behind improvement in circumstances. Social environments can amplify distress through contagion-like mechanisms. By embedding these ideas into a Mesa agent-based model, we can watch how temporary shocks leave lasting traces in a population and how individual trajectories diverge even when everyone lives through the same external events. Mesa’s DataCollector is designed to capture both model-level and agent-level dynamics over time, which makes this kind of temporal analysis especially natural. mesa.readthedocs

20.1 The Logic of Temporal Psychopathology

The core idea of the model is simple: mental health states evolve over time, but they do so through memory rather than instant reaction. Each agent experiences stress from two sources. The first is exogenous stress, generated by scheduled life events such as bereavement, job loss, trauma, and social isolation. The second is endogenous social stress, produced by the presence of symptomatic others in the population. Together, these create the agent’s current stress exposure. Research using agent-based approaches in trauma and mental health has increasingly emphasized that such systems are dynamic, cumulative, and socially embedded rather than reducible to one-off shocks. scholar.afit

The model defines four mental-health states: Healthy, At-Risk, Symptomatic, and Recovering. These states are not merely labels; they structure how agents respond to stress and how likely they are to transition in future steps. To support later quantitative analysis, the code also maps them onto a partial numeric ordering:

HEALTHY     = "Healthy"
AT_RISK     = "At-Risk"
SYMPTOMATIC = "Symptomatic"
RECOVERING  = "Recovering"

STATE_VALUE = {HEALTHY: 0, AT_RISK: 1, RECOVERING: 1.5, SYMPTOMATIC: 2}

This ordering is analytically useful because it allows population-level summaries of final-state dispersion, even though the underlying process remains categorical.

20.2 Stress as Event, Exposure, and Contagion

Stress enters the model through discrete StressEvent objects, each with a severity, duration, and label. These events are scheduled at specific time steps and activated by the model when their trigger time arrives:

class StressEvent:
    def __init__(self, severity: float, duration: int, label: str = ""):
        self.severity = severity
        self.duration = duration
        self.label = label
        self.age = 0

The schedule introduces temporally separated shocks of varying intensity:

STRESS_SCHEDULE = [
    (10, StressEvent(0.6, 5, "Bereavement")),
    (25, StressEvent(0.4, 3, "Job Loss")),
    (40, StressEvent(0.8, 8, "Trauma")),
    (55, StressEvent(0.3, 4, "Social Isolation")),
]

This design matters because it creates a sequence of disturbances rather than one isolated event. A population that has partially recovered from one stressor may still carry residual vulnerability when the next one arrives, opening the door to path dependence. In other words, timing matters, order matters, and history matters.

But external stress is only part of the story. Agents also experience social stress based on the share of others who are currently symptomatic:

def social_stress(self) -> float:
    n_symp = sum(1 for a in self.model.agent_list if a is not self and a.state == SYMPTOMATIC)
    total = self.model.n_agents - 1
    return 0.25 * (n_symp / total) if total > 0 else 0.0

This is a compact but powerful mechanism. It means distress can become self-reinforcing at the population level. When more agents become symptomatic, each individual’s environment grows more stressful, which can push additional agents toward risk or symptom escalation. That mirrors a broader theme in agent-based mental-health research: social systems do not just contain distress, they can propagate it. pmc.ncbi.nlm.nih

20.3 Memory as a Behavioral Filter

The defining feature of the model is memory. Agents do not transition based only on current stress. Instead, they keep a rolling record of recent stress exposure in a fixed-length deque:

self.memory = deque(maxlen=memory_length)

At each step, an agent appends the current stress level to memory, then computes a weighted average of the recent past:

def make_decision(self) -> float:
    recent = list(self.memory)[-5:]
    if not recent:
        return 0.0
    weights = np.exp(np.linspace(0, 1, len(recent)))
    return float(np.average(recent, weights=weights))

The weighting scheme is especially important. Because the weights increase exponentially across the recent window, the most recent stressors matter more than older ones, but older exposures still contribute. This is psychologically plausible and computationally elegant. It captures recency without assuming that only the current moment matters. In effect, the model says that stress leaves a trace, and that trace influences future decisions.

This is where temporal dynamics become more interesting than simple state-switching. Two agents may face identical stress today, but if one has accumulated a heavier recent burden, their effective stress load will be higher. Past exposure reshapes present vulnerability. That is the essence of path dependence.

20.4 Heterogeneity, Resilience, and Vulnerability

Agents are not interchangeable. Each one begins with their own base_resilience and vulnerability, both drawn from seeded random distributions:

self.base_resilience = float(rng.uniform(0.3, 0.9))
self.vulnerability = float(rng.uniform(0.2, 0.9))
self.resilience = self.base_resilience

Mesa models provide seeded randomness through model-controlled generators, which supports reproducibility across runs when the same seed is used. That reproducibility is crucial here because stochastic transitions can otherwise make interpretation difficult. mesa.readthedocs

These two traits serve different conceptual roles. Vulnerability determines how strongly stress exposure affects the agent. Resilience acts as a buffer against deterioration and a resource for recovery. The distinction is subtle but important. Vulnerability captures susceptibility; resilience captures coping capacity. Because both vary across agents, identical environments need not produce identical outcomes.

The model also allows resilience to change over time. During symptomatic periods, resilience erodes slightly:

self.resilience = max(self.resilience - 0.01, 0.05)

During successful recovery back to Healthy, resilience can rebound toward its baseline:

self.resilience = min(self.resilience + 0.05, self.base_resilience)

That means resilience is neither fixed nor infinitely plastic. It is dynamic, but bounded. Experience can wear agents down, and improvement can restore them only partially or gradually. This design gives the model temporal realism by making the consequences of prior states carry forward into future transitions.

20.5 Delays and State Transitions

The transition logic is where memory, vulnerability, resilience, and time spent in state all converge. Each state has its own transition probabilities, and those probabilities are functions of stress load, resilience, vulnerability, and, in some cases, delay.

For example, an initially healthy agent moves to At-Risk with probability:

p_to_risk = 0.03 + 0.90 * stress_load * self.vulnerability * (1 - 0.6 * self.resilience) - 0.05 * delay

Here, stress and vulnerability raise the chance of deterioration, while resilience lowers it. The delay term, based on time already spent in the current state, slightly reduces the transition probability, introducing temporal inertia:

delay = min(self.steps_in_state / 10.0, 0.5)

This is a useful modeling device because it prevents state changes from looking too instantaneous or jittery. Time in state creates stabilization.

The At-Risk state is especially interesting because it allows both escalation and recovery:

p_s = 0.05 + 0.85 * stress_load * self.vulnerability * (1 - 0.5 * self.resilience)
p_h = 0.02 + 0.35 * self.resilience * (1 - stress_load)

This makes At-Risk a genuine transitional zone rather than a waiting room. Agents can tip into symptomatology under sustained burden or return to health if resilience and conditions are favorable. That middle category often matters most in social science models because it is where interventions would be most effective.

For symptomatic agents, recovery becomes more likely as time passes:

p_r = 0.04 + 0.30 * self.resilience * (1 - 0.5 * stress_load) + 0.02 * min(self.steps_in_state, 10)

This is another form of delay logic, but working in the opposite direction. It encodes the idea that symptoms do not necessarily persist forever and that duration itself can alter transition probabilities.

Finally, Recovering agents can either stabilize into health or relapse:

p_h = 0.08 + 0.45 * self.resilience
p_relapse = 0.02 + 0.35 * stress_load * self.vulnerability

This is a particularly realistic feature. Recovery is not represented as a simple reset to baseline. It is a fragile interval in which lingering vulnerability and renewed stress can still produce relapse.

20.6 Model Structure and Data Collection

The MentalHealthModel coordinates event triggering, agent stepping, event aging, and data collection. It uses a custom schedule_step counter to determine when scheduled stressors should be activated:

self.schedule_step = 0

At each model step, the process unfolds in sequence:

  1. Trigger scheduled events whose time has arrived.
  2. Randomize agent order.
  3. Let each agent update once.
  4. Age and remove expired events.
  5. Increment the step counter.
  6. Collect model- and agent-level data.

The randomization of agent order is done with the model’s randomizer:

shuffled_agents = self.agent_list[:]
self.random.shuffle(shuffled_agents)

Mesa supports seeded model randomness, which helps ensure that the same seed reproduces the same shuffle order and stochastic path, provided the rest of the model is unchanged. mesa.readthedocs

The data collection system is one of the model’s major strengths. Mesa’s DataCollector is explicitly designed for model-level and agent-level time series, and your implementation uses both. At the model level, it tracks the fraction of agents in each state, mean resilience, mean cumulative burden, and number of active events. At the agent level, it records each agent’s state, resilience, cumulative burden, and vulnerability. That makes it possible to analyze both aggregate trajectories and individual heterogeneity from the same simulation run. mesa.readthedocs

20.7 Path Dependence in the Results

After running the simulation for 80 steps with 100 agents, the model computes two summary quantities aimed directly at temporal dependence.

The first is a cross-lag correlation between mean cumulative burden at time \(t\) and the symptomatic share at time \(t+5\):

LAG = 5
burden_lagged = df_model["Mean_Burden"].iloc[:-LAG].values
symp_future = df_model["Pct_Symptomatic"].iloc[LAG:].values
corr_lag = float(np.corrcoef(burden_lagged, symp_future)[0, 1])

This is not a causal estimate, but it is a useful diagnostic. If the correlation is strongly positive, it suggests that burden accumulates in ways that foreshadow later symptom prevalence. In other words, population distress may be a delayed consequence of earlier accumulated exposure rather than only of concurrent shocks.

The second summary is a coefficient of variation based on final-state numeric coding:

final_numeric = final_agents["State"].map(STATE_VALUE)
cv = final_numeric.std() / (final_numeric.mean() + 1e-9)

This is a rough indicator of dispersion in end states. It does not fully capture path dependence by itself, but it does reflect how widely agents have diverged after traveling through the same event sequence with different vulnerabilities, resilience levels, and memory-weighted histories. A more path-dependent system should often display stronger divergence in individual outcomes.

20.8 Reading the Visualizations

The visualization suite is especially effective because each panel captures a different temporal layer of the model.

The first panel is a population stackplot of the four mental-health states over time. This gives an immediate sense of regime shifts, such as whether stress events produce temporary spikes in At-Risk agents, sustained increases in Symptomatic agents, or delayed waves of Recovering individuals. Vertical event markers make it possible to compare external shocks with internal state changes.

The second panel tracks mean resilience across the population. This matters because resilience is not fixed. If repeated stressors erode resilience faster than recovery restores it, the entire population may become more fragile over time, even if immediate symptom levels fluctuate.

The third panel plots the cross-lag relationship between cumulative burden and future symptomatic prevalence. By combining scatter with a fitted line, it makes the temporal association visible rather than purely numerical.

The fourth panel shows final state counts, providing a simple end-point summary of the population. While less dynamic than the other plots, it helps anchor interpretation by showing where the system ultimately settles.

The final panel may be the most revealing: individual trajectories for a sample of agents, ordered by vulnerability. This exposes the heterogeneity hidden in population averages. Some agents remain stable despite shocks, some tip into symptoms only after repeated burden, and some oscillate through recovery and relapse. That is exactly what one would expect in a path-dependent system: similar environments, different histories, different outcomes.

20.9 Why This Model Matters

What makes this model compelling is not that it “proves” a theory of psychopathology. Rather, it isolates several mechanisms that social and behavioral scientists often argue are important: accumulated stress, temporal delay, heterogeneity in vulnerability, resilience erosion, recovery, relapse, and socially mediated burden. Agent-based modeling is particularly well suited to this kind of problem because it lets these mechanisms interact over time rather than forcing them into a static equilibrium frame. Reviews of ABM applications in trauma and mental health likewise emphasize its value for representing nonlinear, multilevel, and temporally unfolding processes. scholar.afit

The model also highlights an important conceptual point: not all bad outcomes are caused by the most recent event. Sometimes the decisive factor is the buildup of prior burden. Likewise, recovery is not simply the absence of current stress. It depends on what came before, how resilience has changed, and whether the social environment remains destabilizing. This is what path dependence means in practice. History does not just precede the present; it actively structures it.

20.10 Extensions and Research Directions

This framework opens several promising extensions. One obvious next step is to replace global social stress with network-based contagion, so that distress exposure depends on local peers rather than the whole population. Another is to introduce interventions such as therapy, social support, or income stabilization and examine whether timing matters as much as intensity. Since the model already tracks agent-level burden and vulnerability, it would also be straightforward to test targeted versus universal intervention strategies.

A second extension would be to make memory more psychologically nuanced. Right now, memory is a recency-weighted average of recent stress. One could instead allow salient traumatic events to carry extra weight, introduce forgetting curves, or distinguish between acute and chronic stress. Such changes would deepen the model’s realism while preserving its central temporal logic.

Finally, this model could be used as a teaching bridge between simple Markov-style state models and richer dynamic systems. Its states are discrete and easy to interpret, but its transitions are history-sensitive. That makes it an excellent demonstration of how temporal complexity can emerge even from a relatively compact codebase.

20.11 Closing Perspective

The central lesson of this model is that mental health is not just a matter of current conditions. It is a cumulative, delayed, and socially embedded process. Agents remember stress, carry its effects forward, and respond differently depending on their resilience and vulnerability. Temporary shocks can therefore produce long-lasting consequences, and populations can remain altered long after an event itself has passed.

That is what makes temporal agent-based models so valuable. They help us think beyond snapshots and toward trajectories. In a system with memory, the past is never really gone; it remains encoded in the pathways agents take through time. pmc.ncbi.nlm.nih

import mesa

# ── Cell 3: Constants & State Definitions ────────────────────────────

HEALTHY     = "Healthy"
AT_RISK     = "At-Risk"
SYMPTOMATIC = "Symptomatic"
RECOVERING  = "Recovering"

STATE_VALUE = {HEALTHY: 0, AT_RISK: 1, RECOVERING: 1.5, SYMPTOMATIC: 2}

# ── Cell 4: Stress Event ─────────────────────────────────────────────

class StressEvent:
    def __init__(self, severity: float, duration: int, label: str = ""):
        self.severity = severity
        self.duration = duration
        self.label = label
        self.age = 0

    def is_active(self) -> bool:
        return self.age < self.duration

    def tick(self):
        self.age += 1


STRESS_SCHEDULE = [
    (10, StressEvent(0.6, 5, "Bereavement")),
    (25, StressEvent(0.4, 3, "Job Loss")),
    (40, StressEvent(0.8, 8, "Trauma")),
    (55, StressEvent(0.3, 4, "Social Isolation")),
]

EVENT_STEPS  = [s for s, _ in STRESS_SCHEDULE]
EVENT_LABELS = [e.label for _, e in STRESS_SCHEDULE]

# ── Cell 5: Memory Agent ─────────────────────────────────────────────

class MemoryAgent(mesa.Agent):
    def __init__(self, model, memory_length: int = 10):
        super().__init__(model)

        self.memory = deque(maxlen=memory_length)
        self.state_history = []
        self.decisions_history = []

        rng = model.rng
        self.base_resilience = float(rng.uniform(0.3, 0.9))
        self.vulnerability = float(rng.uniform(0.2, 0.9))
        self.resilience = self.base_resilience

        self.state = HEALTHY
        self.steps_in_state = 0
        self.cumulative_burden = 0.0

    def make_decision(self) -> float:
        recent = list(self.memory)[-5:]
        if not recent:
            return 0.0
        weights = np.exp(np.linspace(0, 1, len(recent)))
        return float(np.average(recent, weights=weights))

    def social_stress(self) -> float:
        n_symp = sum(1 for a in self.model.agent_list if a is not self and a.state == SYMPTOMATIC)
        total = self.model.n_agents - 1
        return 0.25 * (n_symp / total) if total > 0 else 0.0

    def _transition(self, stress_load: float):
        rng = self.model.rng
        rand = float(rng.random())
        delay = min(self.steps_in_state / 10.0, 0.5)

        if self.state == HEALTHY:
            p_to_risk = 0.03 + 0.90 * stress_load * self.vulnerability * (1 - 0.6 * self.resilience) - 0.05 * delay
            p_to_risk = np.clip(p_to_risk, 0, 0.95)
            if rand < p_to_risk:
                self.state = AT_RISK
                self.steps_in_state = 0

        elif self.state == AT_RISK:
            p_s = 0.05 + 0.85 * stress_load * self.vulnerability * (1 - 0.5 * self.resilience)
            p_h = 0.02 + 0.35 * self.resilience * (1 - stress_load)
            p_s = np.clip(p_s, 0, 0.90)
            p_h = np.clip(p_h, 0, 0.90 - p_s)

            if rand < p_s:
                self.state = SYMPTOMATIC
                self.steps_in_state = 0
            elif rand < p_s + p_h:
                self.state = HEALTHY
                self.steps_in_state = 0

        elif self.state == SYMPTOMATIC:
            p_r = 0.04 + 0.30 * self.resilience * (1 - 0.5 * stress_load) + 0.02 * min(self.steps_in_state, 10)
            p_r = np.clip(p_r, 0, 0.80)
            if rand < p_r:
                self.state = RECOVERING
                self.steps_in_state = 0
            self.resilience = max(self.resilience - 0.01, 0.05)

        elif self.state == RECOVERING:
            p_h = 0.08 + 0.45 * self.resilience
            p_relapse = 0.02 + 0.35 * stress_load * self.vulnerability
            p_h = np.clip(p_h, 0, 0.90)
            p_relapse = np.clip(p_relapse, 0, 0.90 - p_h)

            if rand < p_h:
                self.state = HEALTHY
                self.steps_in_state = 0
                self.resilience = min(self.resilience + 0.05, self.base_resilience)
            elif rand < p_h + p_relapse:
                self.state = SYMPTOMATIC
                self.steps_in_state = 0

    def step(self):
        event_stress = sum(ev.severity for ev in self.model.active_events)
        event_stress = min(event_stress, 1.0) * self.vulnerability

        contagion = self.social_stress()
        current_stress = min(event_stress + contagion, 1.0)

        self.memory.append(current_stress)
        self.cumulative_burden += current_stress

        stress_load = self.make_decision()
        self._transition(stress_load)
        self.steps_in_state += 1

        self.state_history.append(self.state)
        self.decisions_history.append(
            (self.model.schedule_step, self.state, stress_load, self.resilience)
        )

# ── Cell 6: Model ────────────────────────────────────────────────────

class MentalHealthModel(mesa.Model):
    def __init__(self, n_agents: int = 100, memory_length: int = 10, seed: int = 42):
        super().__init__(seed=seed)
        self.n_agents = n_agents
        self.memory_length = memory_length
        self.schedule_step = 0

        self.active_events = []

        self._scheduled = [
            (trigger, StressEvent(ev.severity, ev.duration, ev.label))
            for trigger, ev in STRESS_SCHEDULE
        ]

        self.agent_list = []
        for _ in range(n_agents):
            a = MemoryAgent(self, memory_length=memory_length)
            self.agent_list.append(a)

        self.datacollector = mesa.DataCollector(
            model_reporters={
                "Pct_Healthy": lambda m: m._pct(HEALTHY),
                "Pct_AtRisk": lambda m: m._pct(AT_RISK),
                "Pct_Symptomatic": lambda m: m._pct(SYMPTOMATIC),
                "Pct_Recovering": lambda m: m._pct(RECOVERING),
                "Mean_Resilience": lambda m: float(np.mean([a.resilience for a in m.agent_list])),
                "Mean_Burden": lambda m: float(np.mean([a.cumulative_burden for a in m.agent_list])),
                "N_Active_Events": lambda m: len(m.active_events),
            },
            agent_reporters={
                "State": lambda a: a.state,
                "Resilience": lambda a: a.resilience,
                "CumulBurden": lambda a: a.cumulative_burden,
                "Vulnerability": lambda a: a.vulnerability,
            }
        )

        self.datacollector.collect(self)

    def _pct(self, state: str) -> float:
        return sum(1 for a in self.agent_list if a.state == state) / self.n_agents

    def _trigger_events(self):
        for trigger, ev in self._scheduled:
            if trigger == self.schedule_step:
                self.active_events.append(ev)

    def _tick_events(self):
        for ev in self.active_events:
            ev.tick()
        self.active_events = [ev for ev in self.active_events if ev.is_active()]

    def step(self):
        self._trigger_events()

        shuffled_agents = self.agent_list[:]
        self.random.shuffle(shuffled_agents)
        for agent in shuffled_agents:
            agent.step()

        self._tick_events()
        self.schedule_step += 1
        self.datacollector.collect(self)
