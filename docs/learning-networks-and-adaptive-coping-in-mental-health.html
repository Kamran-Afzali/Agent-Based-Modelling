Agent Based Modelling

Chapter 21 Learning, Networks, and Adaptive Coping in Mental Health

In the previous model, mental health unfolded as a temporally structured process shaped by memory, delayed recovery, and path dependence. That framework showed how stress can accumulate and leave residual effects even when the immediate environment improves. The natural next step is to ask what happens when agents are no longer only passive recipients of stress, but active decision-makers embedded in a changing social network. In this extension, mental health becomes not only historical, but adaptive: agents respond to burden by choosing actions, their actions alter exposure, and exposure in turn reshapes future choices.

That shift matters because real people do not simply endure social and clinical environments. They cope, seek help, withdraw, reconnect, and occasionally reorganize their relationships in response to distress. Once those behaviors are introduced, the model ceases to be a one-way stress process and becomes a coupled system of states, actions, and ties. The result is a richer form of temporal dynamics, one in which learning and network structure coevolve alongside mental health trajectories.

21.1 From memory to choice

The earlier model treated recent stress as a weighted internal signal. The present model keeps that idea, but adds agency on top of it. Each agent still carries a memory of exposure, but now that memory helps determine which action the agent selects at each step. Conceptually, the state is no longer just “how stressed is this agent?”, but “how stressed is this agent, what do they see around them, and what can they do about it?”

Formally, we can view each agent as acting in an approximate Markov decision process. Let the observable state be

\[ s_i(t)=\big(x_i(t), b_i(t), r_i(t), c_i(t), n_i(t)\big), \]

where \(x_i\) is clinical state, \(b_i\) is recent burden, \(r_i\) is resilience, \(c_i\) is local symptom exposure, and \(n_i\) is neighborhood position. The point of this representation is not to claim full psychological realism, but to give the agent a finite summary of the environment that is rich enough for learning. A tabular policy can then map these discrete observations to actions without requiring a full continuous control model.

21.1.1 State abstraction

A compact implementation bins the internal and social variables into a small number of levels:

def burden_bin(self):
    avg_recent = np.mean(self.memory) if len(self.memory) > 0 else 0.0
    if avg_recent < 0.20:
        return 0
    elif avg_recent < 0.45:
        return 1
    return 2

def resilience_bin(self):
    if self.resilience < 0.40:
        return 0
    elif self.resilience < 0.70:
        return 1
    return 2

This kind of discretization is crude, but it has a clear advantage: it makes the learning problem interpretable. A low-burden, high-resilience agent is not treated as the same type of learner as a high-burden, low-resilience agent. That distinction is essential if the model is meant to study behavioral adaptation rather than just symptom progression.

21.1.2 Dry run

Suppose an agent has recent burden 0.52, resilience 0.33, and a local symptom fraction of 0.48. Then the agent would occupy a high-burden, low-resilience, high-exposure state. In a tabular learner, that state is precisely the kind of condition under which actions like seeking support or rewiring should become more attractive than passive inaction. The model therefore encodes the intuitive proposition that context-sensitive behavior should emerge under stress, rather than being hard-coded.

21.2 Action as regulation

The key new feature of the model is the action space. Agents can remain passive, cope, seek support, withdraw, or rewire one tie. These are not direct clinical interventions that instantly flip the state from Symptomatic to Healthy. Instead, they are behavioral responses that alter the stress pathway through reduction, buffering, or network modification. That distinction is important because it keeps the simulation plausible: people do not choose a diagnosis, but they do choose or approximate behaviors that shift risk.

A compact action set is therefore:

class Action(IntEnum):
    PASSIVE = 0
    COPE = 1
    SEEK_SUPPORT = 2
    WITHDRAW = 3
    REWIRE = 4

Each action has a cost:

ACTION_COST = {
    Action.PASSIVE: 0.00,
    Action.COPE: 0.08,
    Action.SEEK_SUPPORT: 0.06,
    Action.WITHDRAW: 0.05,
    Action.REWIRE: 0.10,
}

The point of the cost structure is to prevent trivial solutions. If coping were free and always beneficial, every agent would learn to cope at every step. By assigning a small cost, the model forces the learner to weigh short-term effort against longer-term benefit.

21.2.1 Behavioral logic

The action semantics are straightforward. Coping reduces effective stress and gives a small resilience gain. Seeking support reduces stress, improves resilience slightly, and reflects access to positive social contact. Withdrawal lowers contagion but also reduces support, which makes it protective in one dimension and harmful in another. Rewiring modifies the network itself, potentially replacing an unhelpful tie with a more supportive one.

That structure captures a central social-scientific idea: coping behavior is never purely internal. It is entangled with network context. A distressed agent may benefit from support if supportive ties exist, but may also isolate themselves from a harmful environment at the cost of losing buffering resources. The model thus turns a simple clinical process into a decision problem under uncertainty.

21.2.2 Dry run

Imagine two agents with identical burden and resilience. One has a cluster of healthy neighbors; the other is embedded in a symptomatic neighborhood. The first agent may get more value from seeking support, because support is available and support reduces stress. The second agent may benefit more from withdrawal or rewiring, because the surrounding exposure itself is part of the problem. The same state variable can therefore motivate different learned policies depending on local structure.

21.3 Q-learning mechanics

Once actions are available, the model can attach reinforcement learning to the decision layer. The basic update is standard tabular Q-learning:

\[ Q_i(s_t,a_t)\leftarrow Q_i(s_t,a_t)+\alpha\Big[r_t+\gamma \max_{a'}Q_i(s_{t+1},a')-Q_i(s_t,a_t)\Big]. \]

This equation matters because it lets the agent learn from simulated experience rather than from a hand-coded behavioral rule. The model no longer assumes that coping or support-seeking are always taken; instead, it allows their long-run consequences to shape future policy.

21.3.1 Reward design

Reward is the most delicate part of the formulation. If reward only measures clinical state, the learner may ignore meaningful subthreshold dynamics. If reward only measures resilience, the learner may overvalue incremental gains and underweight symptom burden. The implementation therefore combines state value, resilience change, burden penalty, and action cost:

\[ r_i(t)=w_1 H_i(t)+w_2 \Delta R_i(t)-w_3 L_i(t)-w_4 C(a_i(t)). \]

Here \(H_i(t)\) rewards good states, \(\Delta R_i(t)\) rewards resilience gain, \(L_i(t)\) penalizes stress load, and \(C(a_i)\) penalizes costly actions. In code, that becomes:

def compute_reward(self, prev_resilience):
    state_term = STATE_REWARD[self.state]
    burden_penalty = self.model.reward_burden_weight * self._stress_load
    resilience_term = self.model.reward_resilience_weight * (self.resilience - prev_resilience)
    tie_penalty = self.model.reward_isolation_penalty if self.degree() == 0 else 0.0
    action_cost = ACTION_COST[self.last_action]
    reward = state_term + resilience_term - burden_penalty - action_cost - tie_penalty
    return float(reward)

This reward is clinically interpretable. Healthy is better than Recovering, Recovering is better than At-Risk, and Symptomatic is worst. But the model also rewards improvement, not just status. That distinction matters because mental-health recovery often involves gradual movement rather than a binary healed/not-healed cutoff.

21.3.2 Dry run

Suppose an agent ends a step in the Recovering state, increases resilience by 0.03, experiences moderate burden, and chooses to Seek Support. The reward might be modestly positive if the resilience gain and favorable state outweigh the burden and action cost. In contrast, a Symptomatic agent with no resilience gain and a costly action might still receive a negative reward even if the chosen action was adaptive in the long run. That is not a flaw; it reflects the tension between immediate cost and delayed benefit that reinforcement learning is meant to resolve.

21.4 Local social exposure

The earlier version of the model used mean-field contagion: the share of symptomatic agents in the whole population contributed to stress. The new version replaces this with local social exposure on a network. This is a better fit if we want to model support and contagion as neighborhood phenomena rather than global averages. Social systems are rarely well-mixed, and the difference between one’s immediate contacts and the population as a whole often determines how distress spreads or is buffered.

For each agent \(i\), local social exposure can be written as

\[ \sigma_i^{\text{social}}(t)=\beta_c\frac{1}{|N_i(t)|}\sum_{j\in N_i(t)}\mathbf{1}\{x_j(t)=\text{Symptomatic}\} -\beta_s\frac{1}{|N_i(t)|}\sum_{j\in N_i(t)}\mathbf{1}\{x_j(t)\in \{\text{Healthy},\text{Recovering}\}\}. \]

This formulation allows both contagion and support. Symptomatic neighbors contribute upward pressure on stress, while Healthy or Recovering neighbors contribute buffering. In a weighted network, edges can matter unequally, which makes the model more realistic.

def social_signals(self):
    neigh = self.neighbors()
    if not neigh:
        return 0.0, 0.0

    contagion_sum = 0.0
    support_sum = 0.0
    total_w = 0.0

    for other in neigh:
        w = self.edge_weight(other.unique_id)
        total_w += w
        if other.state == SYMPTOMATIC:
            contagion_sum += w
        elif other.state in (HEALTHY, RECOVERING):
            support_sum += w

    if total_w <= 0:
        return 0.0, 0.0

    contagion = self.model.beta_contagion * (contagion_sum / total_w)
    support = self.model.beta_support * (support_sum / total_w)
    return float(contagion), float(support)

21.4.1 Dry run

Take an agent with four neighbors, two Symptomatic and two Recovering, with equal edge weights. Contagion and support may partially cancel. If the symptomatic ties are stronger, the net social signal remains stressful; if the supportive ties are stronger, the same neighborhood may feel protective. The important point is that exposure is no longer just a count of symptoms. It is a structured interaction between local composition and tie strength.

21.5 Dynamic ties

A static network would already improve the model, but the file goes one step further by allowing the network to evolve. Tie weights strengthen when connected agents are in aligned states and weaken when they are mismatched. In addition, some agents may rewire ties based on their current state and social conditions. This makes the network itself a stateful object rather than a fixed background.

The tie-weight update rule is simple:

\[ w_{ij}(t+1)= \begin{cases} \min(w_{ij}(t)+\delta_+, w_{\max}) & \text{if the tie is reinforced},\\ \max(w_{ij}(t)-\delta_-, w_{\min}) & \text{otherwise.} \end{cases} \]

In code, the model treats similar health states as reinforcing and dissimilar states as weakening:

def _update_tie_weights(self):
    edges = list(self.network.edges())
    for u, v in edges:
        au = self.agent_map[u]
        av = self.agent_map[v]
        d = self.network[u][v]

        same_sign = (
            (au.state in (HEALTHY, RECOVERING) and av.state in (HEALTHY, RECOVERING))
            or (au.state == SYMPTOMATIC and av.state == SYMPTOMATIC)
        )

        if same_sign:
            d["weight"] = min(1.5, d.get("weight", 1.0) + self.tie_strengthen)
        else:
            d["weight"] = max(0.05, d.get("weight", 1.0) - self.tie_decay)

        d["age"] = d.get("age", 0) + 1

This creates a mild homophily effect. Similarity increases tie durability; mismatch erodes it. Over time, that can lead to network polarization, with more supportive clusters and more stressed clusters separating from one another. In a mental-health setting, that is not just a structural curiosity. It is a plausible representation of how social environments sort under strain.

21.5.1 Dry run

Suppose two connected agents are both Healthy. Their tie strengthens slightly. Later, if one becomes Symptomatic and the other remains Healthy, that edge begins to decay. If enough such edges weaken, the symptomatic agent may become more isolated or may seek new support through rewiring. The network is therefore not merely a carrier of influence; it is a record of accumulated relational history.

21.6 Rewiring as adaptation

The most interesting action in the new model is rewiring. When an agent chooses to rewire, it drops a relatively poor tie and attempts to connect to a more supportive candidate. This action links learning to network change, and network change to future learning. In other words, the agent is not only choosing what to do, but also choosing who to be near.

The rewiring process is selective. The agent first identifies a current tie to drop, favoring symptomatic, burden-heavy, or weak ties. It then evaluates candidates not already connected, preferring supportive and similar agents. The candidate scoring rule is a compact version of social search:

\[ \text{score}(k)=1.2\cdot \text{similarity}_{ik}+1.4\cdot \text{supportiveness}_k-0.8\cdot \text{burden}_k. \]

This does not make the agent omniscient. It just gives a principled way to prefer better local options.

def rewire_one_tie(self):
    current_neighbors = self.neighbors()
    if len(current_neighbors) <= self.model.min_degree:
        return False

    scored_neighbors = []
    for n in current_neighbors:
        w = self.edge_weight(n.unique_id)
        burden = n.cumulative_burden / max(1, self.model.schedule_step + 1)
        symptom_penalty = 1.0 if n.state == SYMPTOMATIC else 0.0
        score = 1.5 * symptom_penalty + 0.7 * burden - 0.4 * w
        scored_neighbors.append((score, n))

    scored_neighbors.sort(key=lambda x: x[^0], reverse=True)
    drop_agent = scored_neighbors[^0][^1]

The new tie is then selected from among top candidates, with a stochastic weighted choice. This matters because social choice is rarely deterministic. Even a distressed agent with a clear preference for support may not always find the ideal new tie. The model therefore preserves uncertainty while still allowing purposeful adaptation.

21.6.1 Dry run

An agent with several weak symptomatic ties and a few available healthy candidates may drop the worst connection and attach to a more supportive neighbor. On the next steps, that choice changes the agent’s social exposure, the reward signal, and possibly the policy learned by Q-learning. A single rewiring event can thus redirect the trajectory of future burden accumulation.

21.7 Coupled simulation dynamics

The full model is best understood as a coupled process in which health, memory, action, and network structure all influence one another. Each step follows a sequence: trigger events, update tie weights, possibly rewire exogenously, let each agent choose and execute an action, update memory and burden, transition clinical state, compute reward, and update Q-values. The system is deliberately layered so that no single mechanism dominates by construction.

This coupled architecture can be summarized as

\[ x_i(t+1)\sim P_x\big(x_i(t), m_i(t), R_i(t), V_i, N_i(t), a_i(t)\big), \]

\[ m_i(t+1)=\Phi\big(m_i(t), \sigma_i^{\text{event}}(t), \sigma_i^{\text{social}}(t), a_i(t)\big), \]

\[ E_{t+1}\sim P_E\big(E_t, X_t, A_t, W_t\big), \]

\[ Q_i \leftarrow \mathcal{U}\big(Q_i, s_i(t), a_i(t), r_i(t), s_i(t+1)\big). \]

Each equation handles a separate layer: clinical transition, memory update, network evolution, and policy learning. The decomposition is useful because it lets us inspect whether the model changes due to action choice, local exposure, or the coevolution of both.

21.7.1 Dry run

If a stress event occurs while the network is dense with supportive ties, agents may learn coping and support-seeking policies that stabilize them. If the same event occurs in a more polarized or isolated network, withdrawal and rewiring may become more attractive. The model is therefore not just about whether stress is present, but about how the social geometry shapes the policy landscape.

21.8 Reading the results

The visual output of the simulation is designed to track multiple layers of this coevolution. The population stackplot still shows the fraction of agents in each clinical state over time, but now it can be paired with a plot of mean resilience and scaled mean degree. That pairing matters because the network itself is now part of the outcome. A stable population with collapsing social ties is not the same as a stable population with strong support.

The cross-lag burden-to-symptom relationship remains useful as a path-dependence diagnostic, but now it is complemented by the action mix. If coping and support-seeking rise after shocks, the model may exhibit a self-correcting response. If withdrawal and rewiring dominate, the system may show structural adaptation at the cost of reduced cohesion. The distribution of final trajectories also becomes more informative because agents may diverge not only by burden and resilience, but by learned strategy.

21.8.1 Dry run

One useful interpretation is to compare the share of Rewire actions before and after the major trauma event. If rewiring rises after the shock, it suggests that agents are actively restructuring their exposure. If mean degree later falls while resilience improves for some agents, then the network is doing real explanatory work rather than serving as a decorative layer.

21.9 Why this extension matters

This model is valuable because it moves mental-health ABM from passive exposure toward adaptive behavior. That is a major conceptual shift. In many social processes, individuals do not merely absorb context; they respond to it in ways that modify future context. Once that is modeled explicitly, feedback loops become visible.

The learning component also helps bridge ABM and control theory. The agent is no longer only a state machine driven by stochastic transitions. It becomes a learner trying to maximize long-run reward under uncertainty. That makes the model suitable for exploring how different behavioral repertoires may emerge under repeated strain, and how social structure can either support or distort adaptive choice.

21.10 Extensions and next steps

There are several natural extensions. One is to make the network more explicitly weighted and directed, so that support and contagion do not travel symmetrically. Another is to make the action space smaller at first, perhaps by dropping rewiring until the RL layer is validated. A third is to compare this full coevolution model against simpler baselines: a fixed-network model, a learning-only model, and a network-only model.

A more ambitious extension would be to treat the process as a partially observed decision problem rather than a tabular one. The current discretization is useful, but it compresses away many details. A richer state encoder could preserve temporal burden history, tie quality, and perhaps neighborhood composition in a more nuanced way. Even then, the basic insight would remain the same: mental health is not just a function of stress exposure, but of how agents learn to navigate social environments over time.

21.11 Closing perspective

The main lesson of this model is that adaptation and structure belong together. Memory shapes behavior, behavior reshapes network exposure, and network exposure reshapes future memory. In that sense, the system becomes more than a chain of transitions. It becomes a coevolutionary process in which distress, coping, and social context recursively influence one another.

That recursive structure is what makes the model scientifically interesting. It captures not only how mental health changes under stress, but how agents respond to stress in ways that alter the terrain on which later decisions are made. Once that feedback loop is in place, the past no longer acts only through memory. It acts through the social world the agents build around themselves.

21.11.1 MDP design

Formally, define each agent’s decision problem as an MDP \((S, A, P, R, \gamma)\), where the environment is the ABM plus the agent’s own internal history. In this setup, the current implementation already gives you a stochastic transition system, but not yet a true control problem, because agents do not choose actions that affect the transition kernel.1

The first design choice is therefore: what does an agent control? The state should include only variables available to the agent at decision time, while the action should represent a feasible behavioral response such as self-care intensity, help-seeking, avoidance, or social engagement.23

21.11.2 State space

A practical agent state for learning should be a compressed representation rather than the full continuous model state, because tabular Q-learning works best on finite discrete spaces. I would define the observable agent state as

\[ s_i(t)=\big(x_i(t),\, b_i(t),\, r_i(t),\, c_i(t),\, n_i(t)\big) \]

where \(x_i\) is mental-health state, \(b_i\) is recent burden bin, \(r_i\) is resilience bin, \(c_i\) is local contagion/support exposure bin, and \(n_i\) is neighborhood condition such as degree or fraction symptomatic among neighbors.4

A concrete discretization could be:

  • \(x_i \in \{\text{Healthy}, \text{AtRisk}, \text{Symptomatic}, \text{Recovering}\}\)
  • \(b_i \in \{\text{low}, \text{medium}, \text{high}\}\)
  • \(r_i \in \{\text{low}, \text{medium}, \text{high}\}\)
  • \(c_i \in \{\text{low}, \text{high}\}\)
  • \(n_i \in \{\text{isolated}, \text{connected}\}\) or local symptomatic fraction bins.5

This gives a manageable finite state space for tabular Q-learning and preserves the model’s substantive story: current clinical state, accumulated burden, resilience, and local social environment jointly shape behavior.6

21.11.3 Action space

For your model, the most defensible action space is behavioral regulation, not direct “move to Healthy” state transitions. In other words, actions should modify stress processing, resilience, or network interaction probabilities rather than directly set the health state.78

I recommend one of these three action-space designs:

Design Actions Best use
Coping only Do nothing, self-regulate, seek help, withdraw Simplest first version
Social only Maintain ties, seek supportive contact, avoid symptomatic contacts, form new ties Best if network dynamics are central
Hybrid Self-regulate, seek help, socialize, withdraw, rewire ties Most realistic but more complex

A good first implementation is the hybrid but small action set:

  • \(a_0\): passive / no action
  • \(a_1\): coping effort, lowers effective stress load but has a cost
  • \(a_2\): seek support, increases chance of contact with healthy/recovering neighbors and may boost resilience
  • \(a_3\): withdraw, reduces contagion exposure but also reduces support
  • \(a_4\): rewire social tie, replace one neighbor based on a rule.910

That gives real agency without making the model clinically implausible. The action is not “become healthy”; the action is “do something that changes the odds of future trajectories.”11

21.12 Q-learning integration

Once actions exist, the per-agent learner can use Q-learning with the usual update

\[ Q_i(s_t,a_t)\leftarrow Q_i(s_t,a_t)+\alpha\Big[r_t+\gamma \max_{a'}Q_i(s_{t+1},a')-Q_i(s_t,a_t)\Big]. \]

This is appropriate when the transition law is unknown and the agent learns from simulated experience rather than from an explicit model.12

21.12.1 Reward design

The reward is the hardest and most important modeling choice. If reward is badly chosen, the agents will learn odd but mathematically correct behavior.13

Use a reward with four components:

\[ r_i(t)=w_1\,H_i(t)+w_2\,\Delta R_i(t)-w_3\,L_i(t)-w_4\,C(a_i(t)), \]

where:

  • \(H_i(t)\) rewards good mental-health states,
  • \(\Delta R_i(t)\) rewards resilience gain,
  • \(L_i(t)\) penalizes stress burden or symptom severity,
  • \(C(a_i)\) penalizes costly actions such as intense coping or rewiring.14

A concrete mapping:

  • Healthy: \(+2\)
  • Recovering: \(+1\)
  • At-Risk: \(-1\)
  • Symptomatic: \(-3\)
  • Additional penalty: \(-\lambda \times \text{stress\_load}\)
  • Action cost: small negative for coping, help-seeking, and rewiring.15

This prevents degenerate policies like constant rewiring or constant withdrawal. It also reflects that maintaining health matters, but all protective actions are not free.16

21.12.2 Where the action enters

The current model computes stress, stores memory, and then transitions probabilistically. To add learning, insert action choice before transition and let the action alter the transition inputs:

\[ \tilde{\sigma}_i(t)=f\big(\sigma_i^{\text{event}}(t),\,\sigma_i^{\text{social}}(t),\,a_i(t)\big), \]

\[ P\big(x_i(t+1)\mid x_i(t),\tilde{\sigma}_i(t),R_i(t),V_i,N_i(t)\big). \]

Here \(a_i(t)\) changes effective stress, resilience drift, or neighborhood exposure, but the clinical state transition remains stochastic.17

For example:

  • coping reduces effective stress by factor \((1-\eta_c)\),
  • seek support increases resilience by \(+\delta_s\) if supportive contacts are available,
  • withdrawal reduces contagion term but also reduces support term,
  • rewiring changes future neighbors and therefore future social exposure.1819

21.12.3 Pseudocode

For each step t:
    trigger global stress events
    update network if exogenous rewiring occurs

    shuffle agents
    For each agent i:
        observe local state s_i(t)
        choose action a_i(t) using epsilon-greedy policy from Q_i
        apply action effect to:
            effective stress
            resilience drift
            support access
            rewiring probabilities

    For each agent i:
        compute event stress
        compute neighbor contagion/support stress
        update memory buffer
        compute stress_load from memory
        sample next mental-health state x_i(t+1)
        compute reward r_i(t)
        observe next state s_i(t+1)
        update Q_i(s_i(t), a_i(t))

This two-phase update is better than immediate sequential updating because it reduces order effects between action choice and network consequences.2021

21.13 Neighbor interaction

Your current “social contagion” is a mean-field term based on the fraction symptomatic in the whole population. To add neighbors, replace that with a local exposure process on a graph \(G_t=(V,E_t)\), where the edge set may change over time. Dynamic social networks are a natural way to model changing contacts, clustering, and exposure heterogeneity.2223

21.13.1 Network formalism

Let \(N_i(t)\) be the current neighbor set of agent \(i\). Then define social exposure as

\[ \sigma_i^{\text{social}}(t)= \beta_c \cdot \frac{1}{|N_i(t)|}\sum_{j\in N_i(t)}\mathbf{1}\{x_j(t)=\text{Symptomatic}\} \;-\; \beta_s \cdot \frac{1}{|N_i(t)|}\sum_{j\in N_i(t)}\mathbf{1}\{x_j(t)\in\{\text{Healthy},\text{Recovering}\}, \]

possibly with weighted edges \(w_{ij}(t)\). This allows both negative contagion and positive support instead of only harmful exposure.2425

A weighted version is even better:

\[ \sigma_i^{\text{social}}(t)= \sum_{j\in N_i(t)} w_{ij}(t)\,g(x_j(t)), \]

where \(g(\text{Healthy})<0\), \(g(\text{Recovering})<0\), \(g(\text{AtRisk})\approx 0\), and \(g(\text{Symptomatic})>0\).26

21.13.2 What “interaction” should mean

Do not limit interaction to passive contagion. Let an interaction event have one of several consequences:

  • stress contagion,
  • supportive buffering,
  • tie strengthening,
  • tie weakening,
  • information transfer affecting action choice.2728

A simple interaction kernel is:

\[ I_{ij}(t) \sim \text{Bernoulli}(p_{ij}(t)), \]

where \(p_{ij}(t)\) depends on edge weight, homophily, prior interaction, and possibly actions such as “seek support.” If interaction occurs, update stress/support and optionally update edge weight:

\[ w_{ij}(t+1)=w_{ij}(t)+\rho \cdot h(x_i(t),x_j(t),a_i(t),a_j(t)). \]

This lets relationships strengthen or decay endogenously.29

21.14 Dynamic neighbors

The neighbors may change through a rewiring rule. This should be explicit and separate from contagion.3031

21.14.1 Rewiring mechanisms

Use one or more of these:

  • Random turnover: with probability \(p_{\text{rewire}}\), drop one edge and form another uniformly at random.
  • Homophily: agents prefer ties to similar resilience/state/action profiles.
  • Avoidance: healthy or coping agents may sever ties to highly symptomatic neighbors.
  • Support seeking: distressed agents may seek ties to healthy/recovering neighbors.
  • Reciprocity / inertia: strong ties are hard to break.3233

A clean formal rule:

\[ \Pr\big((i,j)\in E_{t+1}\big)\propto \exp\Big( \theta_1 \cdot \text{similarity}_{ij} +\theta_2 \cdot \text{supportiveness}_j -\theta_3 \cdot \text{burden}_j +\theta_4 \cdot \text{prior tie}_{ij} \Big). \]

This turns neighbor change into an adaptive network process rather than an arbitrary graph shuffle.34

21.14.2 Minimal pseudocode for rewiring

For each agent i:
    With probability p_drop(i,t):
        choose current neighbor j to drop
            probability increases if tie weak or contagion high
    With probability p_add(i,t):
        choose candidate k not currently connected
            probability increases with similarity, supportiveness, proximity, prior contact
    add edge (i,k) if degree constraints allow

Degree constraints matter. Without them, the graph can densify unrealistically or collapse into isolates.35

21.15 Recommended implementation order

You should not add everything at once. Add structure in four stages so you can identify what changes the model behavior.3637

21.15.1 Stage 1: Local network without learning

Replace population-wide contagion with neighbor-based contagion on a fixed graph. Keep the current transition equations unchanged except for replacing global symptomatic fraction with local weighted neighbor exposure.3839

21.15.2 Stage 2: Dynamic network

Add rewiring rules, still without Q-learning. This isolates the effect of adaptive neighborhood structure on trajectories and path dependence.40

21.15.3 Stage 3: Q-learning with non-network actions

Add a small action space such as passive, cope, seek support, withdraw. Let actions affect stress load and resilience, but not yet rewiring. This isolates whether learning stabilizes agents even in a static or exogenously changing network.4142

21.15.4 Stage 4: Full coevolution

Let one action affect tie formation or severing, and let network experience affect the learned policy. At this stage you have a coevolving system: health affects ties, ties affect exposure, exposure affects reward, reward updates policy.4344

21.16 Design cautions

The biggest conceptual issue is that the environment is non-stationary because all agents learn simultaneously and the network changes too. Standard tabular Q-learning assumes a stationary environment, so convergence guarantees weaken in this setting. In practice, it can still be useful, but interpret it as adaptive behavior rather than guaranteed optimal control.4546

A second issue is observability. If agents do not observe the full local network or true resilience of neighbors, the problem is closer to a partially observed decision process. You can still use Q-learning with binned observations, but conceptually you are approximating a POMDP with an observed-state surrogate.47

A third issue is calibration. Once actions and network rewiring are added, many parameter combinations can produce similar aggregate curves. Plan from the beginning to run sensitivity analyses over \(\alpha, \gamma, \epsilon\), rewiring probabilities, contagion strength, support strength, and reward weights.48

21.17 Suggested formal specification

I would write the upgraded model as a coupled process:

\[ x_i(t+1)\sim P_x\big(x_i(t), m_i(t), R_i(t), V_i, N_i(t), a_i(t)\big) \]

\[ m_i(t+1)=\Phi\big(m_i(t), \sigma_i^{\text{event}}(t), \sigma_i^{\text{social}}(t), a_i(t)\big) \]

\[ E_{t+1}\sim P_E\big(E_t, X_t, A_t, W_t\big) \]

\[ Q_i \leftarrow \mathcal{U}\big(Q_i, s_i(t), a_i(t), r_i(t), s_i(t+1)\big) \]

where:

  • \(x_i\) is mental-health state,
  • \(m_i\) is memory/burden state,
  • \(E_t\) is the dynamic network,
  • \(A_t\) is the action profile,
  • \(Q_i\) is agent \(i\)’s action-value table.4950

That decomposition will help you keep the implementation modular:

  1. internal stress-memory module,
  2. network interaction module,
  3. clinical transition module,
  4. RL policy module.5152

21.18 Concrete recommendation

For a first serious version, I would use:

  • fixed small discrete state abstraction,
  • 4 actions: passive, cope, seek support, withdraw,
  • weighted local contagion/support on a fixed-degree network,
  • rewiring as an exogenous process first, then as a learned action later.5354

That gets you most of the scientific value without immediately creating an unstable multi-agent RL system. After that, you can compare four models: baseline, network-only, RL-only, and full coevolution. Those comparisons will tell you whether learning changes resilience trajectories, whether adaptive neighborhoods amplify inequality, and whether path dependence becomes stronger or weaker.55

class CoevolvingMentalHealthAgent(mesa.Agent):

        self.memory = deque(maxlen=memory_length)
        self.state_history = []
        self.action_history = []
        self.reward_history = []

        rng = model.rng
        self.base_resilience = float(rng.uniform(0.35, 0.90))
        self.vulnerability = float(rng.uniform(0.20, 0.90))

        self.steps_in_state = 0

        self.last_reward = 0.0

        self._obs_state = None
        self._stress_load = 0.0
        self._event_stress = 0.0
        self._social_stress = 0.0
        self._support_signal = 0.0
        self._effective_stress = 0.0
        self._did_rewire = False

    # Network helpers
    # -------------------------
    def neighbors(self):
        return [self.model.agent_map[n] for n in self.model.network.neighbors(self.unique_id)]

    def degree(self):
        return int(self.model.network.degree[self.unique_id])

    def edge_weight(self, other_id):
        return float(self.model.network[self.unique_id][other_id].get("weight", 1.0))

    # -------------------------
    # RL state abstraction
    # -------------------------
    def burden_bin(self):
        avg_recent = np.mean(self.memory) if len(self.memory) > 0 else 0.0
        if avg_recent < 0.20:
            return 0
        elif avg_recent < 0.45:
            return 1
        return 2

    def resilience_bin(self):
        if self.resilience < 0.40:
            return 0
        elif self.resilience < 0.70:
            return 1
        return 2

    def local_symptom_fraction(self):
        neigh = self.neighbors()
        if not neigh:
            return 0.0
        weighted = 0.0
        total_w = 0.0
        for a in neigh:
            w = self.edge_weight(a.unique_id)
            total_w += w
            if a.state == SYMPTOMATIC:
                weighted += w
        return weighted / total_w if total_w > 0 else 0.0

    def local_symptom_bin(self):
        frac = self.local_symptom_fraction()
        if frac < 0.15:
            return 0
        elif frac < 0.45:
            return 1
        return 2

    def degree_bin(self):
        d = self.degree()
        if d <= self.model.min_degree + 1:
            return 0
        elif d <= self.model.target_degree + 1:
            return 1
        return 2

    def observe_state(self):
        return (
            self.state,
            self.burden_bin(),
            self.resilience_bin(),
            self.local_symptom_bin(),
            self.degree_bin(),
        )

    # -------------------------
    # Q learning
    # -------------------------
    def q_value(self, obs_state, action):
        return self.q[(obs_state, int(action))]

    def choose_action(self):
        s = self.observe_state()
        self._obs_state = s

        if float(self.model.rng.random()) < self.model.epsilon:
            a = Action(int(self.model.rng.integers(0, len(Action))))
            return a

        q_vals = np.array([self.q_value(s, a) for a in Action], dtype=float)
        max_q = np.max(q_vals)
        best_actions = [Action(i) for i, qv in enumerate(q_vals) if np.isclose(qv, max_q)]
        return self.model.random.choice(best_actions)

    def update_q(self, old_state, action, reward, new_state):
        old_q = self.q[(old_state, int(action))]
        future_max = max(self.q[(new_state, int(a))] for a in Action)
        updated = old_q + self.model.alpha * (
            reward + self.model.gamma * future_max - old_q
        )
        self.q[(old_state, int(action))] = float(updated)

    # Social dynamics
    # -------------------------
    def social_signals(self):
        neigh = self.neighbors()
        if not neigh:
            return 0.0, 0.0

        contagion_sum = 0.0
        support_sum = 0.0
        total_w = 0.0

        for other in neigh:
            w = self.edge_weight(other.unique_id)
            total_w += w

            if other.state == SYMPTOMATIC:
                contagion_sum += w
            elif other.state in (HEALTHY, RECOVERING):
                support_sum += w

        if total_w <= 0:
            return 0.0, 0.0

        contagion = self.model.beta_contagion * (contagion_sum / total_w)
        support = self.model.beta_support * (support_sum / total_w)
        return float(contagion), float(support)

        resilience_bonus = 0.0
        support_bonus = 0.0
        self._did_rewire = False

        if action == Action.PASSIVE:
            pass

        elif action == Action.COPE:
            stress *= (1.0 - self.model.cope_stress_reduction)
            resilience_bonus += self.model.cope_resilience_gain

        elif action == Action.SEEK_SUPPORT:
            stress *= (1.0 - self.model.support_stress_reduction)
            support_bonus += self.model.seek_support_bonus
            resilience_bonus += self.model.seek_support_resilience_gain

        elif action == Action.WITHDRAW:
            contagion *= (1.0 - self.model.withdraw_contagion_reduction)
            support *= (1.0 - self.model.withdraw_support_loss)
            stress = max(0.0, min(event_stress + contagion - support, 1.5))
            resilience_bonus -= self.model.withdraw_resilience_penalty

        elif action == Action.REWIRE:
            rewired = self.rewire_one_tie()
            self._did_rewire = rewired
            if rewired:
                support_bonus += self.model.rewire_support_bonus
                stress *= (1.0 - self.model.rewire_stress_reduction)

        support += support_bonus
        self.resilience = float(np.clip(self.resilience + resilience_bonus, 0.05, 1.0))
        stress = max(0.0, min(event_stress + contagion - support, 1.5))

        return float(contagion), float(support), float(stress)

    def rewire_one_tie(self):
        current_neighbors = self.neighbors()
        if len(current_neighbors) <= self.model.min_degree:
            return False

        # choose the "worst" current tie: symptomatic, high burden, weak tie preferred for drop
        scored_neighbors = []
        for n in current_neighbors:
            w = self.edge_weight(n.unique_id)
            burden = n.cumulative_burden / max(1, self.model.schedule_step + 1)
            symptom_penalty = 1.0 if n.state == SYMPTOMATIC else 0.0
            score = 1.5 * symptom_penalty + 0.7 * burden - 0.4 * w
            scored_neighbors.append((score, n))

        scored_neighbors.sort(key=lambda x: x[0], reverse=True)
        drop_agent = scored_neighbors[0][1]

        candidates = []
        existing_ids = {n.unique_id for n in current_neighbors}
        existing_ids.add(self.unique_id)

        for aid, candidate in self.model.agent_map.items():
            if aid in existing_ids:
                continue
            if self.model.network.degree[aid] >= self.model.max_degree:
                continue

            similarity = 1.0 - abs(candidate.resilience - self.resilience)
            supportiveness = 1.0 if candidate.state in (HEALTHY, RECOVERING) else 0.0
            burden_penalty = candidate.cumulative_burden / max(1, self.model.schedule_step + 1)
            score = 1.2 * similarity + 1.4 * supportiveness - 0.8 * burden_penalty
            candidates.append((score, candidate))

        if not candidates:
            return False

        candidates.sort(key=lambda x: x[0], reverse=True)
        top_k = candidates[: min(10, len(candidates))]
        weights = np.array([max(0.01, s + 0.01) for s, _ in top_k], dtype=float)
        weights /= weights.sum()

        idx = int(self.model.rng.choice(np.arange(len(top_k)), p=weights))
        new_agent = top_k[idx][1]

        if self.model.network.has_edge(self.unique_id, drop_agent.unique_id):
            self.model.network.remove_edge(self.unique_id, drop_agent.unique_id)

        self.model.network.add_edge(
            self.unique_id,
            new_agent.unique_id,
            weight=float(self.model.rng.uniform(0.5, 1.0)),
            age=0,
        )

        return True

    # -------------------------
    # Transition dynamics
    # -------------------------
    def make_decision_stress(self):
        recent = list(self.memory)[-5:]
        if not recent:
            return 0.0
        weights = np.exp(np.linspace(0, 1, len(recent)))
        return float(np.average(recent, weights=weights))

    def _transition(self, stress_load: float):
        rng = self.model.rng
        rand = float(rng.random())
        delay = min(self.steps_in_state / 10.0, 0.5)

        if self.state == HEALTHY:
            p_to_risk = (
                0.03
                + 0.90 * stress_load * self.vulnerability * (1 - 0.6 * self.resilience)
                - 0.05 * delay
            )
            p_to_risk = np.clip(p_to_risk, 0, 0.95)
            if rand < p_to_risk:
                self.state = AT_RISK
                self.steps_in_state = 0

        elif self.state == AT_RISK:
            p_s = 0.05 + 0.85 * stress_load * self.vulnerability * (1 - 0.5 * self.resilience)
            p_h = 0.02 + 0.35 * self.resilience * (1 - stress_load)
            p_s = np.clip(p_s, 0, 0.90)
            p_h = np.clip(p_h, 0, 0.90 - p_s)

            if rand < p_s:
                self.state = SYMPTOMATIC
                self.steps_in_state = 0
            elif rand < p_s + p_h:
                self.state = HEALTHY
                self.steps_in_state = 0

        elif self.state == SYMPTOMATIC:
            p_r = 0.04 + 0.30 * self.resilience * (1 - 0.5 * stress_load) + 0.02 * min(self.steps_in_state, 10)
            p_r = np.clip(p_r, 0, 0.80)
            if rand < p_r:
                self.state = RECOVERING
                self.steps_in_state = 0
            self.resilience = max(self.resilience - self.model.symptomatic_resilience_decay, 0.05)

        elif self.state == RECOVERING:
            p_h = 0.08 + 0.45 * self.resilience
            p_relapse = 0.02 + 0.35 * stress_load * self.vulnerability
            p_h = np.clip(p_h, 0, 0.90)
            p_relapse = np.clip(p_relapse, 0, 0.90 - p_h)

            if rand < p_h:
                self.state = HEALTHY
                self.steps_in_state = 0
                self.resilience = min(self.resilience + self.model.recovery_resilience_gain, self.base_resilience)
            elif rand < p_h + p_relapse:
                self.state = SYMPTOMATIC
                self.steps_in_state = 0

    # -------------------------
    # Reward
    # -------------------------
    def compute_reward(self, prev_resilience):
        state_term = STATE_REWARD[self.state]
        burden_penalty = self.model.reward_burden_weight * self._stress_load
        resilience_term = self.model.reward_resilience_weight * (self.resilience - prev_resilience)
        tie_penalty = self.model.reward_isolation_penalty if self.degree() == 0 else 0.0
        action_cost = ACTION_COST[self.last_action]
        reward = state_term + resilience_term - burden_penalty - action_cost - tie_penalty
        return float(reward)

    # -------------------------
    # One step
    # -------------------------
    def step(self):
        prev_resilience = self.resilience
        old_state = self.observe_state()

        action = self.choose_action()
        self.last_action = action

        ev_stress = self.event_stress()
        contagion, support = self.social_signals()

        contagion, support, effective_stress = self.apply_action_effects(
            action, ev_stress, contagion, support
        )

        current_stress = max(0.0, min(effective_stress, 1.0))
        self.memory.append(current_stress)
        self.cumulative_burden += current_stress

        stress_load = self.make_decision_stress()
        self._event_stress = ev_stress
        self._social_stress = contagion
        self._support_signal = support
        self._effective_stress = effective_stress
        self._stress_load = stress_load

        self._transition(stress_load)
        self.steps_in_state += 1

        reward = self.compute_reward(prev_resilience)
        self.last_reward = reward
        new_state = self.observe_state()
        self.update_q(old_state, action, reward, new_state)

        self.state_history.append(self.state)
        self.action_history.append(int(action))
        self.reward_history.append(reward)

class FullCoevolutionMentalHealthModel(mesa.Model):
    def __init__(
        self,
        n_agents: int = 100,
        memory_length: int = 10,
        seed: int = 42,
        target_degree: int = 6,
        small_world_p: float = 0.12,
    ):
        super().__init__(seed=seed)

        self.n_agents = int(n_agents)
        self.memory_length = int(memory_length)
        self.schedule_step = 0

        # learning
        self.alpha = 0.18
        self.gamma = 0.92
        self.epsilon = 0.18
        self.epsilon_min = 0.03
        self.epsilon_decay = 0.992

        # network
        self.target_degree = int(target_degree)
        self.min_degree = max(1, target_degree // 2)
        self.max_degree = max(target_degree + 3, target_degree + 1)
        self.exogenous_rewire_prob = 0.02
        self.tie_decay = 0.01
        self.tie_strengthen = 0.03

        # social dynamics
        self.beta_contagion = 0.35
        self.beta_support = 0.22

        # action effect parameters
        self.cope_stress_reduction = 0.25
        self.cope_resilience_gain = 0.01

        self.support_stress_reduction = 0.12
        self.seek_support_bonus = 0.10
        self.seek_support_resilience_gain = 0.015

        self.withdraw_contagion_reduction = 0.55
        self.withdraw_support_loss = 0.45
        self.withdraw_resilience_penalty = 0.01

        self.rewire_support_bonus = 0.06
        self.rewire_stress_reduction = 0.08

        # clinical drift
        self.symptomatic_resilience_decay = 0.01
        self.recovery_resilience_gain = 0.05

        # reward
        self.reward_burden_weight = 1.6
        self.reward_resilience_weight = 6.0
        self.reward_isolation_penalty = 0.3

        self.active_events = []
        self._scheduled = [
            (trigger, StressEvent(ev.severity, ev.duration, ev.label))
            for trigger, ev in STRESS_SCHEDULE
        ]

        # build network
        if self.target_degree >= self.n_agents:
            raise ValueError("target_degree must be < n_agents")
        if self.target_degree % 2 == 1:
            self.target_degree += 1  # Watts-Strogatz prefers even k

        self.network = nx.watts_strogatz_graph(
            n=self.n_agents,
            k=self.target_degree,
            p=small_world_p,
            seed=seed
        )

        for u, v in self.network.edges():
            self.network[u][v]["weight"] = float(self.rng.uniform(0.5, 1.0))
            self.network[u][v]["age"] = 0

        # agents
        self.agent_list = []
        self.agent_map = {}
        for i in range(self.n_agents):
            a = CoevolvingMentalHealthAgent(self, memory_length=self.memory_length)
            self.agent_list.append(a)
            self.agent_map[a.unique_id] = a

        # map agent IDs to graph nodes by relabeling graph if needed
        # assumes unique_id order aligns to creation order in Mesa
        graph_nodes = list(self.network.nodes())
        uid_map = {graph_nodes[i]: self.agent_list[i].unique_id for i in range(self.n_agents)}
        self.network = nx.relabel_nodes(self.network, uid_map)

        self.datacollector = mesa.DataCollector(
            model_reporters={
                "Pct_Healthy": lambda m: m._pct(HEALTHY),
                "Pct_AtRisk": lambda m: m._pct(AT_RISK),
                "Pct_Symptomatic": lambda m: m._pct(SYMPTOMATIC),
                "Pct_Recovering": lambda m: m._pct(RECOVERING),
                "Mean_Resilience": lambda m: float(np.mean([a.resilience for a in m.agent_list])),
                "Mean_Burden": lambda m: float(np.mean([a.cumulative_burden for a in m.agent_list])),
                "Mean_Degree": lambda m: float(np.mean([m.network.degree[a.unique_id] for a in m.agent_list])),
                "Mean_TieWeight": lambda m: m.mean_tie_weight(),
                "Pct_Action_Cope": lambda m: m._pct_action(Action.COPE),
                "Pct_Action_SeekSupport": lambda m: m._pct_action(Action.SEEK_SUPPORT),
                "Pct_Action_Withdraw": lambda m: m._pct_action(Action.WITHDRAW),
                "Pct_Action_Rewire": lambda m: m._pct_action(Action.REWIRE),
                "Mean_Reward": lambda m: float(np.mean([a.last_reward for a in m.agent_list])),
                "N_Active_Events": lambda m: len(m.active_events),
            },
            agent_reporters={
                "State": lambda a: a.state,
                "Resilience": lambda a: a.resilience,
                "CumulBurden": lambda a: a.cumulative_burden,
                "Vulnerability": lambda a: a.vulnerability,
                "Action": lambda a: ACTION_LABELS[a.last_action],
                "Reward": lambda a: a.last_reward,
                "Degree": lambda a: a.degree(),
                "LocalSymptomFrac": lambda a: a.local_symptom_fraction(),
            }
        )

        self.datacollector.collect(self)

    # -------------------------
    # Metrics
    # -------------------------
    def _pct(self, state: str) -> float:
        return sum(1 for a in self.agent_list if a.state == state) / self.n_agents

    def _pct_action(self, action: Action) -> float:
        return sum(1 for a in self.agent_list if a.last_action == action) / self.n_agents

    def mean_tie_weight(self):
        if self.network.number_of_edges() == 0:
            return 0.0
        return float(np.mean([d.get("weight", 0.0) for _, _, d in self.network.edges(data=True)]))

    # -------------------------
    # Events
    # -------------------------
    def _trigger_events(self):
        for trigger, ev in self._scheduled:
            if trigger == self.schedule_step:
                self.active_events.append(ev)

    def _tick_events(self):
        for ev in self.active_events:
            ev.tick()
        self.active_events = [ev for ev in self.active_events if ev.is_active()]

    # -------------------------
    # Dynamic network
    # -------------------------
    def _update_tie_weights(self):
        edges = list(self.network.edges())
        for u, v in edges:
            au = self.agent_map[u]
            av = self.agent_map[v]
            d = self.network[u][v]

            same_sign = (
                (au.state in (HEALTHY, RECOVERING) and av.state in (HEALTHY, RECOVERING))
                or (au.state == SYMPTOMATIC and av.state == SYMPTOMATIC)
            )

            if same_sign:
                d["weight"] = min(1.5, d.get("weight", 1.0) + self.tie_strengthen)
            else:
                d["weight"] = max(0.05, d.get("weight", 1.0) - self.tie_decay)

            d["age"] = d.get("age", 0) + 1

    def _exogenous_rewire(self):
        for a in self.agent_list:
            if float(self.rng.random()) < self.exogenous_rewire_prob:
                a.rewire_one_tie()

    # -------------------------
    # Step
    # -------------------------
    def step(self):
        self._trigger_events()
        self._update_tie_weights()
        self._exogenous_rewire()

        shuffled_agents = self.agent_list[:]
        self.random.shuffle(shuffled_agents)
        for agent in shuffled_agents:
            agent.step()

        self._tick_events()
        self.schedule_step += 1
        self.epsilon = max(self.epsilon_min, self.epsilon * self.epsilon_decay)
        self.datacollector.collect(self)
